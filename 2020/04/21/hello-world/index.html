Hello World
发表于 2020-04-21| 更新于 2020-04-22
阅读量:

第一次见面，惯例不能破，致敬“hello world!”

第一篇文章，想写写关于我是怎么把这个网站从零开始到现在这个模样的。

​ 最开始是想着写一个个人博客，日常记录一些专业知识和突然的灵光一闪加一些想法，那时候只玩弄过云服务器＋域名的方式，就想着阿里云买台服务器+域名，然后利用宝塔搭建一个wordpress个博。然后在域名备案的这段时间呢突然在网上看到别人的一个像个博一样的网站，并且网址是XXX.github.io，这网址让我产生了好奇，然后一步一步的发现这是搭建在github上的一个网站，然后引出了hexo这么个神奇的博客平台。经过两天的摸索，从最初的再github 上搭建hexo，填了几个坑成功搭建出hexo后，发现进网页速度有时候挺慢，在网上寻找解决方案的时候看到有加CDN的，有搭建在别的平台上的（码云或者coding等），然后继续摸索，码云说是不会自动更新什么的，最后还是选择了coding。不过github也留着，似乎并不冲突，我的理解就是自己本地弄弄弄然后发到github或者coding平台而已，有一个或者多个问题不大。后来也看到有说，配置域名解析的时候国内访问coding海外访问github这样配置，“自己想的果然没错”。

基础环境：

  • 能上互联网的正常电脑、安装好git、node.js等。
  • 安装好git后，需要对git进行用户名和密码配置。
    1. $ git config –global user.name “Your Name”
    2. $ git config –global user.email “email@example.com“

hexo方面：

  • 在文件夹中git bash here，然后下载hexo

  • hexo init #初始化，该文件夹会生成许多文件

  • hexo install #安装依赖包

  • hexo常用三个命令：

    1. hexo clean #清除生成的静态文件
    2. hexo g #你理解为更新就好
    3. hexo d #上传你更新的
    4. hexo s #本地测试用，会开启hexo并占用4000端口。

  • 一般要改的地方就两个，都在hexo根目录下的_config.yml文件里：

    1. URL里面，有时候会改url和root（啥时候我也不知道，反正这坑我是踩了，最后root没改就放了个/）

    2. 最下面，deploy要改，里面有type: ‘git’，repo: 这里加你的平台和连接方式，比如：github: https连接方式或者ssh连接方式，然后还有个是branch: master

  • 为了能上传到github或者coding上面，我们需要配置SSH key。此方式相比较HTTPS可以不需要每次都验证用户名和密码。下面介绍如何配置SSH Key：

    1. 创建一个SSH Key：ssh-keygen -t rsa -C “username”
    2. 按三次回车，然后会在你的用户文件夹下生成key文件。具体位置在系统盘/User/你的用户（一般Administrator）/ssh/
    3. 然后把文件里面的东西全部复制，粘贴到平台的公钥上面。

平台方面：

​ 无论是github还是coding，首先都要注册好账号。
